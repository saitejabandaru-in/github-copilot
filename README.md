## Core Concept: Comment-Driven Development

Copilot reads your comments and context to generate code.
The better your comments, the better the code.

Write intent in comments BEFORE writing code:
- Describe what the function should do
- List expected inputs and outputs
- Specify edge cases to handle
- Note performance requirements

## Copilot Chat Commands

- `/explain` — Explain selected code
- `/fix` — Fix bugs in selected code
- `/test` — Generate unit tests
- `/doc` — Generate docstrings
- `/optimize` — Suggest performance improvements

## Advanced Techniques

### Type Hint Guidance
Strong type hints guide Copilot to correct implementations.
Always add type hints before asking Copilot to implement.

### Example-Driven Completion
Show Copilot input/output examples in comments.
It will match the pattern for your actual input.

### Intent Comments
Specify constraints: 'Use vectorized operations, not loops'
Specify error handling: 'Raise ValueError for invalid dates'
